## Phase 1: デザインの格を上げる（コード品質ではなくデザイン品質）

### 1-1. スタイリング統一
- **全ページをBEMスコープドCSSに統一**（ホームのインラインstyle全廃）
- **Footerをinline/BEMに**（現在Tailwindクラスのみで他と不統一）
- **ホームのCompanyInfoを About/Contact と完全統一**

### 1-2. タイポグラフィ精密化
- Foster+Partners参考: Open Sans + 単色アクセント（黄1色）
- APDW方針: Syne(Display) + Outfit(Body) + Noto Sans JP は維持
- **見出しの行長制御**（max-width: 20ch 等で改行位置をコントロール）
- **本文の行長制御**（max-width: 65ch で可読性確保）
- **キャプション・補助テキストのサイズを0.8rem以上に統一**

### 1-3. 余白・レイアウトの一貫性
- **全セクションのpadding規則を統一**（現在ページごとにバラバラ）
- **セクション間の区切り方を統一**（border-top? 余白のみ? 罫線?）
- **コンテンツ幅の使い分けを定義**
  - フルブリード: ヒーロー画像、Featured
  - ナロー（720px）: テキスト本文、Quote
  - ワイド（1440px）: グリッド、カード

### 1-4. 画像品質
- **全AI生成画像に img-graded + img-grain を適用**（一部漏れあり）
- **srcset / width / height を全画像に追加**（CLS対策 + パフォーマンス）
- **ヒーロー画像を各ページ固有に**（完了済み、OG画像対応済み）

## Phase 2: サービスフロー視覚化（AI ページ）

### 2-1. ビジネス構造（確定）
```
入口（並列）:
  セミナー — 単発OK。サークル会員は月額内で受講可能
  サークル — 全ツール使い放題 + セミナー受講権。月額5,000円
      ↓
組織展開:
  法人研修 — 共通言語化。ハンズオン70%
      ↓（大企業・余裕ある組織）
  導入プログラム — 3〜6ヶ月。業務分析→構築→定着
      ↓
定着:
  保守顧問 — 研修後 or 導入PG後。月1面談 + チャットサポート

必要時:
  受託開発 — 既存ツールの限界時。建築実務を知る開発チーム
```
- セミナー→サークル入会の導線あり
- 2A(研修)→2B(導入PG)の遷移可能性あり
- 保守顧問は研修後にも導入PG後にも来る

### 2-2. 視覚化の方針
- **カルーセル内で段階を伝える**（別セクションに図を足さない）
- 参考: Stripe Enterprise の progressive disclosure
- 参考: Powerhouse Company の progress visualization
- ジャーニーインジケーターで段階を示し、分岐はインジケーター上で表現
- **顧客の状態（困りごと）をサービス名より先に出す**

### 2-3. 実装アプローチ
- ジャーニーインジケーターのドットを構造化（入口/展開/定着/実装の4フェーズ）
- 分岐点でドットが二股に分かれるビジュアル
- 保守顧問は「研修の後にも導入PGの後にも来る」ことを示すために、複数の線がここに収束する表現
- 超一流参考サイトのレベルで仕上げる（ボックス+矢印は禁止）
