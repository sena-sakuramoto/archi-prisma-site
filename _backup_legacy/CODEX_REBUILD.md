## 技術スタック

### 現在の構成
- **フレームワーク**: Astro 5（静的サイトジェネレーター）
- **スタイリング**: Tailwind CSS v4（@tailwindcss/vite プラグイン経由）
- **デプロイ**: GitHub Pages（GitHub Actions で自動デプロイ）
- **ドメイン**: archi-prisma.co.jp（CNAME ファイルで設定済み）
- **パッケージマネージャ**: pnpm

### 技術選定は自由に変更してよい

フレームワーク・ホスティングともに、**より良い選択肢があれば変更してOK**。

**フレームワークの選択肢:**
- **Astro（現状）**: 静的サイトには最適。コンテンツ駆動、高速、JS最小。建築事務所のコーポレートサイトとして妥当
- **Next.js**: SSR/ISRが必要なら。ただしこのサイトは完全静的なのでオーバースペック
- **素のHTML + Tailwind**: 最もシンプル。ビルドツールが少なくメンテしやすい
- **その他**: 合理的な理由があれば何でもOK

**ホスティングの選択肢:**
- **GitHub Pages（現状）**: 無料、シンプル、カスタムドメイン対応。静的サイトなら十分
- **Cloudflare Pages**: 無料、エッジ配信、ビルド速い。お問い合わせフォームにWorkers使える
- **Vercel**: Astro/Next.jsとの相性良し。無料枠あり
- **Netlify**: フォーム機能内蔵。無料枠あり

**判断基準**: このサイトに必要なのは「速くて信頼性のある静的ホスティング」のみ。動的機能は不要。コスト最小で安定動作するものを選ぶこと。

**変更する場合**:
- 新しい構成でのセットアップ手順も含めること
- デプロイのCI/CD設定も書き直すこと
- CNAME / DNSの設定変更が必要なら手順を書くこと

## 現在の問題点（なぜリビルドするか）

### デザインの問題
1. **テック企業っぽすぎる** — 建築設計事務所なのにSaaSスタートアップのようなデザイン。グロー・シマーアニメーション、パララックスエフェクトが建築ブランドに合わない
2. **英語が多すぎる** — ナビが全部英語（About, Brand, Services...）。日本の建築クライアント向けなのに不自然
3. **BrandPanelセクションが不要** — カラーパレットやタイポグラフィのショーケースはデザインシステムのドキュメントであってコーポレートサイトに置くものではない
4. **情報過多** — セクションが多すぎて何が重要か分からない（Hero, Brand, Services, Works, Partners, Team, Contact = 7セクション）
5. **StatsRibbon が嘘っぽい** — "1000+ Projects" は誇大。プロジェクト数は60件程度
6. **CTAが不明確** — "Project Desk" が何を意味するか分からない
7. **動画ヒーローが重い** — 5.3MBの動画がLCPを悪化させる

### 構造の問題
1. コンテンツが `content/` ディレクトリにmarkdown/JSONで分離されているのは良い → **この構造は維持**
2. `src/lib/content.ts` でコンテンツを読み込む仕組みも維持
3. コンポーネント分割は良いが、各コンポーネントが肥大化している

## 新デザインの方向性

### コンセプト
**「建築を語る、建築のサイト」**
- 建築事務所のサイトは**作品（Works）が主役**。華美なアニメーションではなく、プロジェクトの写真と実績で語る
- 日本語メインでクライアント（デベロッパー、事業オーナー、個人施主）に信頼感を与える
- ミニマルで余白を活かしたデザイン。建築的な美しさ

### トーン
- **落ち着いた知性**: テック感ではなく、プロフェッショナルな設計事務所の空気
- **バイリンガル**: ナビ・セクション見出しは日本語メイン、英語はアクセント程度に
- **写真中心**: テキストより画像で魅せる。プロジェクト写真を大きく使う

### カラーシステム（global.css は自由に変更してOK）
現在の primary（青系）+ surface（スレート系）は維持しつつ、以下を調整:
- 背景は白ベース（`#ffffff` or `#fafafa`）— 現在の `surface-50` でOK
- テキストは濃いグレー/黒 — `surface-800` 〜 `surface-900`
- アクセントは控えめに — `primary-500` を差し色として
- **ゴールド/真鍮系のアクセント**を追加検討（建築的な高級感）

### タイポグラフィ
- **見出し**: Barlow Condensed は維持（建築的でクリーン）
- **本文**: Noto Sans JP は維持
- **数字・年号**: Barlow Condensed で大きく表示（建築ポートフォリオ的な演出）
